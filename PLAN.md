## [PROSE]

This plan outlines the transition of the "Academic Vibing" corpus from its current "Pre-preprint" state (v0.3) to a formal "Preprint" (v0.4) and eventual submission to arXiv. The focus of this iteration is on technical hardening and empirical validation of the methodology's core claims.

### 1. Technical Development: Automated CDI Kit
The primary objective is to move the **Consensus Divergence Index (CDI)** from a manual calculation to an automated diagnostic tool.
- **Task:** Develop scripts/agents to parse multi-agent outputs and calculate divergence based on semantic overlap and claim contradictions.
- **Outcome:** A standardized CDI report generated for every major research synthesis phase.

### 2. Empirical Validation: The 3-Arm Test
To move beyond auto-ethnographic proof, we will implement the 3-arm validation framework proposed in earlier issues.
- **Arm A (Control):** Traditional human-only research.
- **Arm B (Pure AI):** Un-scaffolded multi-agent synthesis.
- **Arm C (Vibing):** RAC network with CDI and human vibe-checking.
- **Goal:** Quantify the speed-rigor trade-off across all three arms.

### 3. Structural Hardening: Cross-Model Consensus Audit
To address the **Collusive Consensus Fallacy**, we will conduct an audit of the current corpus using frontier models with fundamentally different training sets and alignment paradigms (e.g., DeepSeek, Grok, Llama 3) to identify hidden systemic biases.

### 4. Final Submission Pipeline
- **Zenodo Archiving:** Ensure all version-controlled receipts have stable DOIs.
- **arXiv Formatting:** Convert the dual-section Markdown (Rock + Prose) into a standardized LaTeX/PDF format suitable for arXiv submission.
- **GitHub Issue-Loop Closure:** Resolve all remaining [BLOCKER] tags in the current issue loop before release.
